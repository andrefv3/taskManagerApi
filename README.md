# Tasks App API

A scalable and production-ready Task Management API built with modern backend technologies and best practices.

## 📌 Overview

This service provides a robust RESTful API for managing tasks, authentication, and user workflows.  
It is designed with a modular architecture to ensure maintainability, scalability, and clarity — following patterns used in real-world production systems.

## Project setup

```bash
$ npm install
$ npx prisma generate
$ npm build
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev
```

## 🚀 Features

- RESTful API design
- JWT-based authentication and authorization
- Full task lifecycle management (CRUD)
- Task filtering by status (pending, in-progress, completed)
- Database integration using Prisma ORM
- Environment-based configuration

## 🧠 Architecture & Engineering

- Modular, feature-based architecture
- Strong typing with TypeScript
- Separation of concerns (controllers, services, modules)
- Scalable backend structure inspired by enterprise systems
- Centralized error handling and validation

## 🛠 Tech Stack

- NestJS
- TypeScript
- Prisma ORM
- PostgreSQL
- JWT (Authentication)

## 🌍 Deployment

Deployed as a cloud-based service using Render.

## 📈 Purpose

This project demonstrates backend engineering skills including API design, authentication, database modeling, and clean architecture — aligned with industry standards used in scalable systems.
